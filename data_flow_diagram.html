Level 1 — Process Decomposition

🔐
P1
User Authentication
  • IN Registration form (name, email, phone, password)
  • IN Login credentials (email, password)
  • OUT Hashed password stored → DS: users
  • OUT Session token / Flask-Login cookie
  • OUT Flash message (success / error)
🔍
P2
Browse Services & Workers
  • IN HTTP GET / (homepage)
  • IN GET /api/workers?service=…
  • IN GET /api/search?q=…
  • OUT Services list → index.html
  • OUT Top-6 workers (by rating) → index.html
  • OUT Recent 3 reviews → index.html
  • OUT JSON worker data → browser
📅
P3
Book a Service
  • IN POST /book (service, worker_id, address, city, date, time)
  • IN current_user.id (from session)
  • OUT New Booking row → DS: bookings
  • OUT Status = "pending"
  • OUT Flash "Booking confirmed"
  • OUT Redirect → homepage
📋
P4
View My Bookings
  • IN GET /my-bookings (login required)
  • IN current_user.id (session)
  • OUT Query bookings filtered by user_id
  • OUT Ordered by booking_date DESC
  • OUT Rendered → bookings.html
⚡
P5
REST API Layer
  • IN GET /api/services
  • IN GET /api/workers
  • IN GET /api/search?q=
  • OUT JSON: [{id, name, icon, description}]
  • OUT JSON: [{id, name, service, rating, …}]
  • OUT JSON search results
